Kubernetes 배포

Kubernetes 클러스터가 실행되면 컨테이너 응용 프로그램을 그 위에 배치 할 수 있습니다. 이렇게하려면 Kubernetes 배포 구성을 만듭니다. 배치는 Kubernetes에게 지시합니다. 응용 프로그램의 인스턴스를 만들고 업데이트하는 방법. 배포를 만들면 Kubernetes 마스터 스케줄은 클러스터의 개별 노드에 어플리케이션 인스턴스를 언급합니다.

응용 프로그램 인스턴스가 생성되면 Kubernetes Deployment Controller는 해당 인스턴스를 지속적으로 모니터링합니다. 인스턴스를 호스팅하는 노드가 중지되거나 삭제되면 배치 제어기가 이를 대체합니다. 이는 기계 고장 또는 유지 보수 문제를 해결하는자가 치유 메커니즘을 제공합니다.

사전 오케스트레이션 세계에서는 설치 스크립트가 종종 응용 프로그램을 시작하는 데 사용되지만 컴퓨터 오류로부터의 복구는 허용되지 않았습니다. Kubernetes Deployments는 응용 프로그램 인스턴스를 작성하고 노드간에 실행되도록 유지함으로써 응용 프로그램 관리에 근본적으로 다른 접근 방식을 제공합니다.

당신의 첫번째 앱을 Kubernetes에 배포하세요.


Kubernetes 명령 줄 인터페이스 인 Kubectl 을 사용하여 배포를 만들고 관리 할 수 ​​있습니다. Kubectl은 Kubernetes API를 사용하여 클러스터와 상호 작용합니다. 이 모듈에서는 Kubernetes 클러스터에서 애플리케이션을 실행하는 배치를 작성하는 데 필요한 가장 일반적인 Kubectl 명령을 학습합니다.

배포를 만들 때 응용 프로그램의 컨테이너 이미지와 실행할 복제본 수를 지정해야합니다. 나중에 배치를 업데이트하여 해당 정보를 변경할 수 있습니다. 모듈 5 및 6 는 배포를 확장하고 업데이트 할 수있는 방법을 설명합니다.

응용 프로그램은 Kubernetes에 배포하기 위해 지원되는 컨테이너 형식 중 하나로 패키지화해야합니다.

첫 번째 배포에서는 Docker 컨테이너에 패키지화 된 Node.js 응용 프로그램을 사용합니다. 소스 코드와 Dockerfile은 Kubernetes Bootcamp의 GitHub 저장소 에서 사용할 수 있습니다.

이제 Deployments가 무엇인지 알았으므로 온라인 자습서로 이동하여 첫 번째 앱을 배포하십시오!
